## 一句话主线

Z1+ 属于 topological analysis / primitive path analysis 工具族。它要解决的问题是：给定线性聚合物链坐标和盒子，在固定链端、不允许链穿越、路径长度单调缩短的约束下，找到 shortest multiple disconnected path (SMDP)，并输出 primitive path 网络、contour length `Lpp`、kink/entanglement 数 `Z`、以及若干 `Ne` estimator。

这条路线有两个分支：

- 几何分支：Z / Z1 / Z1+。把链当作无体积但不可穿越路径，用 local geometric operations 缩短 contour length。优点是 parameter-free、速度快、可以定位 kink 的空间和化学位置。
- 动力学分支：standard PPA / PPA+。通过关掉 intrachain excluded volume、保留 interchain excluded volume、FENE/WCA 类势和低温动力学收缩得到 PP 网络。优点是概念接近传统 PPA，缺点是会有 chain sliding / loss of entanglements 风险，也不能像几何算法一样自然输出 binary entanglement 对。

## 2005 Kroger CPC：原始 Z-code / SMDP

当前直接证据：

- Z1+ README 把这篇列为 `Zcode` 参考，题名为 *Shortest multiple disconnected path for the analysis of entanglements in two- and three-dimensional polymeric systems*，CPC 168, 209-232, 2005。
- CPC/Mendeley 数据页说明原始程序名是 `Z`，catalogue id `ADVG_v1_0`，目标是对 2D/3D polymer configuration 取得 primitive paths / shortest multiple disconnected path。
- 数据页还说明 PP 被定义为连接固定链端、尊重拓扑约束的 shortest line/path；2D 解唯一，3D 在构造意义下有唯一性条件。

算法意义：

- 这是 Z1+ 的祖先。Z1+ 2023 论文明确说 Z1+ 是 Z 和 Z1 的 successor。
- 2005 的核心思想不是做动力学能量最小化，而是直接求几何意义上的 SMDP。
- 后续 2009 IJMS 和 2023 CPC 对它的可操作描述是一致的：固定链端、禁止 bond crossing / chain interpenetration、单调缩短 contour length，最终形成由 straight segments 连接的 disconnected polygons / primitive paths。

对 Python 改造的影响：

- 如果没有原始 Z / Z1 几何核心源码，Python 版的 default Z1+ 不能简单“翻译 Fortran”；只能按 2005/2009/2023 论文做 clean-room 复现，然后用原二进制 benchmark 做 black-box 校验。
- 必须先把 SMDP 约束写成测试契约：端点不动、每条链 contour length 不增加、链段不得穿越、输出的 kink/other-chain 对与原程序在 benchmark 上近似一致。

## 2007 Shanbhag & Kroger Macromolecules：annealing PPA 与几何方法的差异

当前直接证据：

- Z1+ README 把这篇列为 `Shanbhag2007`。
- 作者页摘要说明：这篇批判比较了取得 primitive path network 的现有方法，连接原始 annealing 方法和几何方法；well-entangled polymers 的平均 PP 长度存在约 15% discrepancy，主要归因于 annealing 过程中发生 disentanglement；ring polymer toy configurations 显示约一半差异在没有 end-looping 时仍存在，归因于 slip；在 chain diameter 很小且 rapid quenching 的极限，annealing 方法可实践上接近几何方法。

算法意义：

- 这篇不是 Z1+ 的实现说明，而是方法学警告：PPA 和 Z1/Z1+ 都称为 primitive path / SMDP 分析，但它们的误差结构不同。
- 标准 PPA 通过能量最小化找近似 SMDP，几何 Z1 直接最小化路径长度；前者可能因为 chain sliding / disentanglement 丢失拓扑约束。
- 2023 Z1+ 论文在介绍 PPA+ 时也提醒，energy minimizers 本身不保证防止链解缠、滑移或穿越。

对 Python 改造的影响：

- 如果先做 Python MVP，PPA/PPA+ 比 default Z1+ 更容易，因为公开包里有 `module-PPA.f90` 和 `module-CPPA.f90`；但它不能替代几何 Z1+。
- 验证时不能只比较 `Lpp`；如果目标是 entanglement topology，还要比较 `Z1+SP.dat` 中 kink 位置、`Z_values.dat`、以及 `other-chain/other-node` 对。

## 2009 Hoy/Foteinopoulou/Kroger PRE：Ne estimator 与链长效应

当前直接证据：

- arXiv 摘要明确指出：单链长 `N` 上工作的 `Ne` estimator 会有系统性 `O(1/N)` 误差，根源包括把 chain ends 当成 entanglements，以及忽略 chain / primitive path dimensions 的非 Gaussian 修正。
- 本地 PDF 文本抽取确认论文反复讨论 direct enumeration of entanglements、M-coil、M-kink、chain-end effects、non-Gaussian primitive path statistics。
- 论文提出的多个 chain length 的 M-estimator 能显著改善收敛；direct enumeration / kink-based formulas 更简单、通常更快收敛。

关键概念：

- `S-estimators`：只用单个 `N` 或单个 configuration 的统计量。Z1+ 默认 summary 中报告的 `NeCK`, `NeMK`, `NeCC`, `NeMC` 都属于此类。
- `M-estimators`：使用多个 chain length 或 homologous series，把 `N -> infinity` 的变化显式纳入。它们更准确，但不能从单个 configuration 无假设地给出。
- `Z` / kink-based estimator 与 `Lpp` / coil-based estimator 的物理假设不同。coil-based estimator 依赖 PP conformational statistics，kink-based estimator 更接近 direct enumeration。

与 Z1+ 输出的对应：

- Z1+ 2023 Appendix A 明确继承这套 estimator 体系。
- `Z1+summary.dat` 的 15 列中，`NeCK`, `NeMK`, `NeCC`, `NeMC` 对应 classical S-kink、modified S-kink、classical S-coil、modified S-coil。
- Z1+ 不能从单个 configuration 自动返回 M-estimator；但可以通过多个 `Z1+summary.dat` 后处理得到。

对 Python 改造的影响：

- Python 版不应只追求输出 `Lpp`，还要准确保留 per-chain `N`, `Ree`, `Lpp`, `Z`, `LPP`, density 等 summary 基础量，否则未来无法复现 M-estimator。
- estimator 层可以较早用 Python 实现，因为公式相对独立；但它依赖前端几何/PPA 核心给出的 `Z` 与 `Lpp` 是否可信。

## 2009 Karayiannis & Kroger IJMS：Z1 workflow 和物理节点解释

当前直接证据：

- MDPI open access 文章说明它讨论从 atomistic polyethylene 到 freely jointed tangent hard sphere packings 的建模、平衡和 primitive path 后处理。
- Section 4.1 直接描述 Z1：固定 chain ends，关闭 excluded volume 同时保留 uncrossability，通过几何变换单调减少 contour length，返回 PP network、`Lpp`、interior kinks `Z`。
- 文章强调 periodic images of the same chain 被视为不同链，connected parts 仍视作同一链；self-entanglements 默认忽略。
- Section 4.2 区分 mathematical path 和 physical primitive path：算法内部可有很多零长度或几乎直线节点，但物理路径要移除不改变方向的节点和零长度段；剩下的内部节点叫 interior kinks。

关键理解：

- Z1 的内部节点数不是物理量；物理量是 path shape、total contour length、以及清理后的 kink。
- `Z` 是 primitive path 上方向改变次数，与“有多少条其他链造成该 entanglement”不是一一对应。一个 kink 至少涉及其他链阻挡，但多条链也可能共同造成一个 kink。
- PPA vs Z1 的主要差异包括：PPA 可能 loss of entanglements、最小化 energy 而非 length、速度慢；几何方法速度快几个数量级，且基本 parameter-free。

对 Python 改造的影响：

- Python 版需要显式区分 algorithmic nodes 与 physical nodes。不能把每个中间节点直接当作 entanglement。
- kink 清理规则需要可测试，例如角度阈值、零长度阈值、节点合并后的 contour length 不变性。
- `selfZ` / self-entanglements 应作为明确选项，而不是混入默认路径。

## 2023 Kroger/Dietz/Hoy/Luap CPC：Z1+ 包本身

当前直接证据：

- ETH Research Collection 显示该文 open access，2023-02，CPC 283, article 108567，license 为 CC BY 4.0。
- 论文摘要说明 Z1+ 是 Z/Z1 successor，新增 adaptive neighbor lists、保持较多 temporary nodes 以改善大系统性能，并提供 Java GUI 与 PPA+。
- Program summary 给出 license Apache-2.0，语言 Fortran 90 / Perl / Java，外部需求 Perl 5+ 与 JRE 1.8+。
- 论文把问题形式化为：输入多链线性 polymer configuration 与盒子，寻找满足固定链端、禁止 bond crossing、path length 单调减少的 SMDP。
- 论文给出安装、14 个内置 benchmark、Z1/LAMMPS/Z1+ 格式、summary 输出和 `Z1+SP.dat` 格式。

实现与输出：

- public distribution 文件清单与本地目录吻合：`Z1+install.pl`, `Z1+template.pl`, README, shared/output/folding/messages/PPA/CPPA/main modules, import/rearrange scripts, `Z1+.ex`, `Z1+.jar`, benchmarks, LICENSE/NOTICE。
- 输入支持 Z1 snapshot/trajectory、LAMMPS data、LAMMPS dump、Z1+ snapshot/trajectory。Z1+ 格式在 Z1 基础上可加入 sheared/non-cubic periodic boxes 和 label。
- `Z1+SP.dat` 每条链记录 shortest path nodes；用 `+` / `-SP+` 选项时会额外输出 responsible `other-chain` 和 `other-node`。
- `Z1+summary.dat` 一行 15 个量：`label C N Ree Lpp Z app bpp LPP NeCK NeMK NeCC NeMC b0 n`。PPA/PPA+ 输出 `PPA-summary.dat` / `PPA+summary.dat`，但 `Z` 未定义时设为 `-1`。
- Appendix A 汇总 `Ne` estimator：Z1+ 自动计算 S-estimator；M-estimator 需要 homologous series 后处理。

PPA/PPA+ 警告：

- Z1+ 论文明确区分 geometrical Z1+ 和 dynamical PPA/PPA+。
- standard PPA 得到的是 energy minimum 附近的 SMDP，可能不适合识别 individual entanglements。
- Z1+ 几何方法可给出 entanglement point 的空间位置和沿链化学位置，所以对 anisotropic / deformed systems 更有价值。

## Python 改造建议

建议分层，不要一上来承诺完整 Z1+ 几何核心等价：

1. `z1io`：读取/写入 Z1/Z1+/LAMMPS data/dump 的 Python parser；输出统一 dataclass / numpy arrays。
2. `z1summary`：实现 `Ree`, `Lpp`, `Z`, `app`, `bpp`, `LPP`, `NeCK`, `NeMK`, `NeCC`, `NeMC` 的 post-processing；这层由论文公式和 README 直接支撑。
3. `z1runner`：封装 Linux `Z1+.ex`，在集群上批量跑 benchmark 和用户输入，收集标准输出与 artifact。这个阶段即可实用。
4. `ppa_py`：按公开 `module-PPA.f90` / `module-CPPA.f90` 移植 PPA/PPA+，先对小 benchmark 做 `PPA-summary.dat` 级别对齐。
5. `geom_z1_py`：clean-room 实现几何 Z1+。这一层最难，需要把 local geometric operations、neighbor candidate selection、temporary nodes、crossing tests、node cleanup、binary entanglement pairing 都拆成可测模块。

验收顺序：

- parser round-trip 不改变 benchmark 坐标/链长/盒子。
- summary estimator 用已知 `Z1+summary.dat` 行做数值回归。
- wrapper 在 Linux 集群和本地 Mac 的行为分开：Mac 只做解析和远程提交，不直接执行 ELF。
- PPA/PPA+ 先允许数值近似误差；default Z1+ 几何输出需要更严格比较 `Z`, `Lpp`, kink count 和 `SP+` pairing。

## 当前结论

我们有足够资料做 Z1+ 的工程化 Python 外壳、输入/输出 parser、summary estimator、以及 PPA/PPA+ 移植；但没有足够公开源码直接翻译 default Z1+ 几何算法。完整 Python 版可行，但应按 clean-room + black-box regression 的路线推进。论文已经给出了核心约束和输出语义，集群上的 Linux 二进制可以作为行为基准。
